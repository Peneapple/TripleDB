Sustainability and stable identifiers

Version history and update policy

A dedicated release page makes maintenance visible, preserves old downloads and documents how source changes affect existing TDB records.

Proposed cadence

Planned at least annually

Each update compares the new BioGRID release with the previous source release, reviews newly added and modified candidate records, and applies the same curation rules.

Stable accession policy

TDB IDs are not reassigned

Old links remain resolvable. Revisions, reclassifications and deprecations are recorded rather than hidden.

Change categories

What every release should report

Added

New records

New TDB accessions and the source evidence that triggered inclusion.

Revised

Annotation changes

Same accession; corrected roles, phenotype, condition or provenance.

Reclassified

Interpretation changes

Relation category or evidence confidence changed after review.

Deprecated

Withdrawn records

Accession remains visible with reason, date and replacement link.

i
Source-release comparison should include more than new IDs.

Also inspect modified metadata and source records that were corrected, merged or removed.
