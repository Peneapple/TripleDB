Purpose and governance

About TripleDB

TripleDB is a manually curated, versioned resource for higher-order genetic relationships that are difficult to represent as isolated pairwise interactions.

Scope

The core object is a curated multi-gene relationship supported by primary literature and, where applicable, one or more BioGRID source interaction records.

  • Triple and higher-order gene combinations
  • Symmetric and directional genetic relationships
  • Phenotype, condition and qualification context
  • Stable accessions and transparent source links

Design principles

  • Free access without login
  • Evidence visible from every record
  • Structured metadata plus original text
  • Versioned downloads and stable URLs
  • Correction history instead of silent replacement
  • Mobile-readable and accessible interface
Citation

How to cite TripleDB

Please cite the versioned TripleDB release when using downloaded data. For record-level citation, include the stable TDB accession, release version and access date. The database manuscript citation will be added after publication.

Peter X. Geng and Huaiqiu Zhu. TripleDB: a curated database of triple-gene and higher-order genetic interactions. Version 1.0.0. Zenodo. https://doi.org/10.5281/zenodo.20931612.

Version DOI: 10.5281/zenodo.20931612 · Website: https://tripledb.org

Community input

Suggested record submissions

Suggested records should include organism, gene combination, relation type, PMID/DOI, source location and evidence text. Submission and correction history will be reviewed before inclusion in a public release.

Policies

Data access, licensing and maintenance

Data license

TripleDB curated annotations and release files are distributed under the Creative Commons Attribution 4.0 International License. External source records, identifiers and publications should also be cited according to the original source database and publication requirements.

Privacy

TripleDB search does not require registration or login. The static website reads public JSON files in the browser and does not require users to submit account information.

Maintenance

Stable releases are archived through GitHub and Zenodo. Updates are planned at least annually, with change summaries provided in the version history page.
